# 支线《以毒攻毒》调用接口

## 启动剧情

```js
window.SideStoryAPI.start('poison_counter_poison');
```

默认会播放 `src/data/side_story_poison_counter_poison.json` 的第一个场景，并在结束后返回 `GameScene`。

如果要从某个场景索引开始：

```js
window.SideStoryAPI.start('poison_counter_poison', { sceneIndex: 0 });
```

如果只是预加载支线数据：

```js
const story = await window.SideStoryAPI.loadStory('poison_counter_poison');
```

## 地图/NPC 接入建议

王大壮 NPC 的建议 ID：

```js
wang_dazhuang
```

NPC 交互时调用：

```js
window.SideStoryAPI.start('poison_counter_poison');
```

剧情完成后会设置：

```js
side_poison_counter_poison_complete
```

最近一次医学问答结果会写入：

```js
window.gameStateManager.state.__medicalQuizResult
```

## 新增通用小游戏

剧情 step 类型：

```json
{
  "type": "minigame_medical_quiz",
  "quiz": {
    "type": "single",
    "title": "题目标题",
    "prompt": "题干",
    "options": [
      { "id": "A", "text": "选项A" },
      { "id": "B", "text": "选项B", "correct": true }
    ],
    "explanation": "解析文本"
  }
}
```

已支持：

| quiz.type | 用途 |
| --- | --- |
| `single` | 单选题 |
| `multi` | 多选题 |
| `matching` | 连线题 |
| `herbSelect` | 组方选药题 |

## 已修复的逻辑问题（2026-05-28）

### 1. 背景切换缺失
**问题**：整个支线始终使用 `bg_village_center.png`，从村口→王家内室无视觉过渡。
**修复**：`SPCP_wang_001`（推开王大壮家的门）新增 `"backgroundOverride": "bg_wang_home_interior"`，进入内室后自动切换背景。

### 2. SPCP_end_003 步骤类型错误
**问题**：`"type": "dialogue"` + `"character": "narrator"` 触发了对话UI逻辑，但 narrator 映射到 📖 emoji，与语义不匹配。
**修复**：改为 `"type": "narration"`，规范为居中旁白文本。

### 3. setFlag 在 narration 步骤中不生效
**问题**：`_showNarration()` 未处理 `step.setFlag`，导致 `side_poison_counter_poison_complete` 永远不被设置。
**修复**：在 `_showNarration()` 中新增 `if (step.setFlag) { this._setFlag(step.setFlag, true); }`。
> 修复范围：`IntroScene.js` 第 1180 行附近。

### 4. 支线 NPC 头像未预加载
**问题**：`npc_wang_dazhuang.png` 和 `npc_wang_laotai.png` 不在 `_loadCharacterImages()` 列表中，即使文件存在也无法显示。
**修复**：新增 `sideStoryCharacters` 预加载数组，包含这两张头像。素材缺失时以 Emoji 兜底（💪/🤒）。

### 5. Emoji 兜底缺失
**问题**：`wang_dazhuang` / `wang_laotai` 不在 `_getCharacterEmoji()` 映射中，素材缺失时显示 ❓。
**修复**：新增 `'wang_dazhuang': '💪'`、`'wang_laotai': '🤒'`。

### 6. SPCP_final_choice 出现太突然（2026-05-28 晚）
**问题**：`SPCP_end_001` 主人公说"你这个问题问得很好"后直接弹出三个选项评价走方郎中，无任何过渡。
**修复**：新增 `SPCP_end_001b`（旁白：王大壮拿出纸片对比两张方子）和 `SPCP_end_001c`（王大壮提问："他是不是在骗我？"），自然引出选项。

### 7. 奖励 UI 简陋（2026-05-28 晚）
**问题**：`_showReward()` 仅显示纯文本 `获得：\n★ xx\n...`，无布局和动画。
**修复**：重写 `_showReward()`，新增深色圆角面板、金色装饰、圆形图标卡片、品质标签（图鉴解锁/声望）、逐件弹入动画。支持 `step.title` 和 `step.displayDuration` 配置。
