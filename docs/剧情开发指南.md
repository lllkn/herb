## 一、项目架构总览

```
Game1/
├── src/
│   ├── IntroScene.js          # 剧情场景引擎（核心，2145行）
│   ├── data.js                # 静态数据定义（HERBS_DATA, ATTRIBUTES_DATA 等）
│   ├── gameState.js           # 游戏状态管理 + GameAttr 全局接口
│   └── ui.js                  # HTML UI 管理（背包/图鉴/属性面板等）
├── public/
│   └── src/assets/
│       ├── pictures/          # 原始大图
│       │   ├── 学堂背景立绘.png / 师傅立绘1.png / 小兰.png / 灵宠.png ...
│       │   ├── 甘草特写.png / 菊花特写.png / 鹅卵石特写.png ...
│       │   └── 药圃背景.png / 药房.png / 炮制间.png / 过道.png ...
│       └── pictures/          # Phaser 加载用图（分目录）
│           ├── backgrounds/   # 背景图 bg_xxx.png
│           ├── characters/    # 角色头像/表情 bai_smile.png 等
│           └── prologue/      # CG 大图 cg_01~cg_04.png
├── index.html                 # 主页面（弹窗HTML在此）
├── styles.css                 # 全局样式（含属性面板等）
└── src/data/story_prologue.json  # 序章剧情数据（母版）
```

### 核心类关系

```
IntroScene (Phaser.Scene)
├── _executeStep()     → 核心调度引擎，按 step.type 分发
├── TypewriterSystem   → 打字机效果
├── ChoiceSystem       → 选择分支UI
├── TutorialOverlay    → 教程遮罩引导
└── CGDisplaySystem    → CG大图演出

window.gameStateManager → 状态管理单例
├── addHerbToBackpack()    → 添加草药到背包（自动解锁图鉴）
├── initAttributes()       → 初始化属性
├── addAttribute()         → 单项增减属性
└── batchAddAttributes()  → 批量增减属性

window.GameAttr → 全局便捷接口（剧情脚本专用）
├── .add(id, delta)        → 单项增加
├── .addMany(...)          → 批量增加
├── .get(id)               → 查询值
└── .getAll()              → 全部快照

window.uiManager → UI刷新
├── updateBackpackUI()     → 刷新背包
├── updateHerbGuideUI()    → 刷新图鉴
└── updateAttributesUI()   → 刷新属性面板
```

## 二、剧情 JSON 数据模板

### 2.1 文件结构

```json
{
  "version": "1.0",
  "lastUpdated": "2026-XX-XX",
  "description": "章节描述",
  "storyId": "chapter_01",
  "title": "章节标题",
  "scenes": [
    { /* 场景1 */ },
    { /* 场景2 */ }
  ]
}
```

### 2.2 场景节点（Scene）定义

```jsonc
{
  "id": "S03",                    // 唯一标识，格式: S + 数字
  "name": "岐黄学堂·清晨",        // 中文名称（仅作注释）
  "type": "dialogue",             // 场景类型:
                                  //   "monologue" - 黑屏旁白(如S01)
                                  //   "scene"     - 有背景图场景(如S02)
                                  //   "dialogue"  - 对话场景(如S03)
  "duration": 180,                // 预估时长(秒)，仅供参考
  "background": "bg_school_yard.png", // 背景图文件名(对应 pictures/backgrounds/)
  "bgm": "bgm_school.mp3",        // BGM文件名(TODO:暂未实现)
  "steps": [ /* 步骤数组，见下文 */ ]
}
```

**背景图命名规则**：
| 用途 | 文件名示例 | 放置路径 |
|------|-----------|---------|
| 学堂庭院 | `bg_school_yard.png` | `pictures/backgrounds/` |
| 药圃 | `bg_herb_garden.png` | `pictures/backgrounds/` |
| 炮制间 | `bg_processing_room.png` | `pictures/backgrounds/` |
| 药房 | `bg_spirit_room.png` | `pictures/backgrounds/` |

> 新增背景图需在 `_loadBackgroundImages()` 方法中注册加载。

### 2.3 步骤类型一览

`_executeStep()` 核心调度器支持的 **step.type**：

| type | 处理方法 | 说明 |
|------|---------|------|
| `narration` | `_showNarration()` | 屏幕居中旁白文本 |
| `dialogue` | `_showDialogue()` | 对话框对话（含头像+立绘+特写） |
| `choice` | `_showChoice()` | 选择分支（多选项→汇聚） |
| `tutorial` | `_showTutorial()` | 教程遮罩引导 |
| `stepAlias` | 直接跳转 | 分支汇聚点，跳转到 aliasTo |
| `cg` / `cg_display` | `_showCG()` | CG大图演出 |
| `pet_selection` | `_showPetSelection()` | 灵宠选择 |
| `reward` | `_showReward()` | 奖励发放（背包+物资+属性） |
| `title_card` | `_showTitleCard()` | 标题卡（大字淡入淡出） |
| `end` | `_endPrologue()` | 结束序章 |

### 2.4 各步骤字段详解

#### narration（旁白）

```jsonc
{
  "id": "S01_001",
  "type": "narration",
  "text": "大草纪，中历三百四十七年秋。",
  "textColorStart": "#8a7050",    // 可选，文字渐变起始色
  "textColorEnd": "#8a7050",      // 可选，文字渐变结束色
  "bgColor": "#000000",           // 背景色（纯色场景用）
  "sound": null,                  // 音效（TODO）
  "nextDelay": 2000,              // 自动继续延迟(ms), 500=点击继续
  "triggerFadeIn": true,          // 可选,触发淡入下一场景
  "nextScene": "S02"              // 可选,下一步跳转到指定场景
}
```

#### dialogue（对话）

```jsonc
{
  "id": "S03_001",
  "type": "dialogue",
  "character": "bai",             // 角色ID: bai/xiaolan/qingmiao/player/narrator
  "expression": "smile",          // 表情: smile/thoughtful/serious/happy/anxious/grateful
  "name": "白院长",               // 显示名称
  "text": "孩子，在这学堂，你已勤修三载。",
  "avatar": "bai_smile.png",      // 头像文件名(pictures/characters/)
  "overlayImage": "gancao_closeup", // 可选,特写图片textureKey
  "hidePortrait": false,          // 可选,true则隐藏角色立绘
  "nextDelay": 500,
  "setFlag": "flag_name",         // 可选,设置游戏标记
  "nextScene": "S04",             // 可选,跳转场景
  "unlockHerb": "gancao"          // 可选,自动解锁该草药图鉴
}
```

**character ↔ avatar ↔ emoji 映射**：

| character | 头像前缀 | emoji | 立绘纹理 |
|-----------|---------|-------|---------|
| `bai` | `bai_` | 👴 | `bai_standing` |
| `xiaolan` | `xiaolan_` | 👧 | `xiaolan_standing` |
| `qingmiao` | `qingmiao_` | 🌱 | `lingchong_portrait` |
| `player` | — | 😊 | null |
| `narrator` | — | 📖 | — |

**overlayImage（特写图片）使用方式**：

在 dialogue step 中添加 `"overlayImage": "图片key"` 即可。连续多个 step 使用同一 overlayImage 时不会重复播放动画。

特写图片需要：
1. 在 `preload()` 中通过 `this.load.image('key', 'path')` 注册
2. 在 JSON 中引用 key 名

#### choice（选择分支）

```jsonc
{
  "id": "S03_choice1",
  "type": "choice",
  "options": [
    {
      "text": "选项A文案",
      "flag": "confident",           // 标记玩家选择(存入 gameState.flags)
      "response": {
        "type": "dialogue",          // 回应类型(通常为dialogue)
        "character": "bai",
        "expression": "smile",
        "name": "白院长",
        "text": "回应文案...",
        "avatar": "bai_smile.png",
        "nextDelay": 500,
        "nextStep": "S03_004"        // ⚠️ 必填! 汇聚到同一后续步骤ID
      }
    },
    {
      "text": "选项B文案",
      "flag": "humble",
      "response": {
        "character": "bai",
        "expression": "smile",
        "name": "白院长",
        "text": "不同回应...",
        "avatar": "bai_smile.png",
        "nextDelay": 500,
        "nextStep": "S03_004"        // 所有选项必须汇聚!
      }
    }
  ],
  "nextDelay": 0                     // choice的nextDelay通常为0
}
```

**⚠️ 选择分支关键规则**：
1. 每个 option 的 `response.nextStep` 必须指向同一个后续步骤 ID（用于多路汇聚）
2. `flag` 字段会存入 `gameState.flags`，可用于后续条件判断
3. 选项数量建议 2~4 个

#### reward（奖励发放）

```jsonc
{
  "id": "S04_009",
  "type": "reward",
  "items": [
    { "id": "gancao", "name": "甘草", "count": 1, "icon": "🌿" },
    { "id": "juhua", "name": "菊花", "count": 2, "icon": "🌼" },
    { "id": "copper", "name": "铜钱", "count": 200, "icon": "💰" },
    { "id": "recommendation", "name": "院长推荐信", "count": 1, "icon": "📜" },
    { "id": "reputation", "name": "声望", "count": 10, "icon": "⭐" }  // 特殊:不入库,仅展示
  ],
  "nextDelay": 1200
}
```

**奖励处理逻辑** (`_showReward`)：

| id 类型 | 处理方式 | 目标存储 |
|---------|---------|---------|
| 匹配 HERBS_DATA 中的 id | `gameStateManager.addHerbToBackpack()` | `state.backpack` (同时解锁图鉴) |
| `copper` | 直接加到 `state.copper` | `state.copper` |
| `reputation` / `achievement` | **跳过不入库**，仅显示文本 | 无 |
| 其他任意 id | 写入 `state.inventory[id]` | `state.inventory` |

奖励显示后会自动调用 `uiManager.updateBackpackUI()` 和 `uiManager.updateHerbGuideUI()` 刷新UI。

#### CG演出（CG大图）

```jsonc
{
  "id": "S05_awakening",
  "type": "cg",
  "cgId": "CG_03",
  "cgConfig": {
    "image": "cg_03_awakening.png",  // 图片文件名
    "fadeInDuration": 800,            // 淡入时长(ms)
    "animation": "awakening",         // 动画类型
    "petId": "qingmiao"               // 关联灵宠(可选)
  },
  "nextDelay": 1000
}
```

CG也可带文字旁白：
```jsonc
{
  "id": "S09_cg",
  "type": "cg",
  "cgId": "CG_04",
  "cgConfig": { "image": "cg_04_departure.png" },
  "text": "（画面渐远……前方，山川延绵。）",  // CG上叠加旁白
  "nextDelay": 3000
}
```

#### tutorial（教程遮罩）

```jsonc
{
  "id": "S04_tutorial_1",
  "type": "tutorial",
  "tutorialAction": "show_herb_guide",  // 教程动作类型
  "hint": "按 T 打开本草图谱",         // 提示文字
  "hintPosition": "top-right",         // 提示位置
  "nextDelay": 500
},
{
  "id": "S04_tutorial_5",
  "type": "tutorial",
  "tutorialAction": "collect_juhua",   // 采集教程
  "herbId": "juhua",
  "requiredCount": 2,
  "hint": "去采集菊花×2（靠近后按E），进度：0/2",
  "nextDelay": 1000
}
```

#### title_card（标题卡）

```jsonc
{
  "id": "S09_title",
  "type": "title_card",
  "text": "第 一 章",
  "subText": "寒窑村·初至",
  "duration": 2500,
  "nextDelay": 1000
}
```

#### end（结束）

```jsonc
{
  "id": "S09_end",
  "type": "end"
}
```
触发 `_endPrologue()`：保存状态 → 进入 GameScene 主游戏。

### 3.2 图片加载注册方式

新增任何图片后，必须在 `IntroScene.preload()` 中注册：

```javascript
// ========== 背景图 ==========
_loadBackgroundImages() {
    const bgImages = [
        'bg_school_yard', 'bg_herb_garden', 'bg_processing_room',
        'bg_spirit_room', 'bg_dean_room', 'bg_school_gate'
    ];
    bgImages.forEach(key => {
        this.load.image(key, `src/assets/pictures/backgrounds/${key}.png`);
    });
}

// ========== 角色/表情头像 ==========
_loadCharacterImages() {
    const characters = [
        'bai_smile.png', 'bai_thoughtful.png', 'bai_serious.png',
        'xiaolan_anxious.png', 'xiaolan_grateful.png',
        'qingmiao_surprised.png', 'qingmiao_happy.png'
    ];
    characters.forEach(file => {
        this.load.image(file.replace('.png',''), `src/assets/pictures/characters/${file}`);
    });
    
    // 灵宠立绘
    this.load.image('lingchong_portrait', 'src/assets/picture/灵宠.png');
    // 灵宠精灵图（粒子）
    this.load.image('pet_qingmiao', 'src/assets/pictures/characters/pet_qingmiao.png');
}

// ========== CG大图 ==========
_loadCGImages() {
    ['cg_01_school', 'cg_02_spirit_room', 'cg_03_awakening', 'cg_04_departure']
        .forEach(key => this.load.image(key, `src/assets/pictures/prologue/${key}.png`));
}

// ========== 特写图片（按需注册）==========
// 在 preload() 中单独添加：
this.load.image('gancao_closeup', 'src/assets/picture/甘草特写.png');
this.load.image('juhua_closeup', 'src/assets/picture/菊花特写.png');
this.load.image('eshi_closeup', 'src/assets/picture/鹅卵石特写.png');
```

### 3.3 图片格式要求

| 用途 | 推荐格式 | 尺寸建议 | 备注 |
|------|---------|---------|------|
| 背景图 | PNG | 1920×1080 或更大 | 会缩放适配屏幕 |
| 角色立绘全身 | PNG（透明底） | 高度≥1080px | 透明PNG，人物居中偏下 |
| 表情头像 | PNG（透明底或方形） | 76×76px 显示区域 | 圆形裁切显示 |
| CG大图 | JPG/PNG | 1920×1080 | 全屏展示 |
| 特写图片 | PNG | 400×300 显示区 | 右侧显示，带金色边框 |
| 灵宠精灵 | PNG（透明底） | 小尺寸 | 用于粒子动画 |

## 四、UI 元素位置与尺寸

以下坐标基于 **1280×720 设计分辨率**（Phaser 默认），实际运行时按比例自适应。

### 4.1 对话框

```
┌───────────────────── 1280px 宽度 ─────────────────────┐
│                                                       │
│                    （立绘区域）                         │
│                   depth: 8                            │
│                                                       │
│  ┌─────────────────────────────────────────┐          │
│  │ ┌────┐  名称文字(黄色26px)              │ ← depth:10
│  │ │头像│  对话内容文字(白色24px)            │   半透明黑底85%不透明度
│  │ │76x6│  wordWrap宽度=对话框宽-168        │   金色圆角边框
│  │ └────┘                                   │   圆角16px
│  │                                          │   位置: 底部(y=height-250)
│  └─────────────────────────────────────────┘          │
│              宽度 = (width-100) × 3/4                  │
│              高度 = 200                                │
│  ▸ 点击继续 提示 (右下角,闪烁动画)                      │
└───────────────────────────────────────────────────────┘
```

**关键参数**：
- 对话框宽度：`(screenWidth - 100) * 3 / 4` ≈ 885px（@1280宽）
- 对话框高度：200px
- 对话框 Y 坐标：`screenHeight - 250`
- 头像框：76×76px，位于对话框内左上角偏移(16, 16)
- 头像 Emoji：36px，居中于头像框
- 名称文字：黄色(#ffcc00)、26px、仿宋体
- 内容文字：白色、24px、仿宋体、lineSpacing=8
- 继续提示：右下角 `(width-80, height-70)`、14px、闪烁动画

### 4.2 角色立绘

```
┌──────────────────────────────────────────┐
│                                          │
│            ┌────────────┐                │
│            │            │                │
│            │  立绘图片   │  depth: 8      │
│            │  锚点居中   │  位于屏幕中央  │
│            │  position:  │  (w/2, h*0.5) │
│            │  (w/2,     │  初始极小:     │
│            │   h*0.5)   │  20×32 px     │
│            │            │                │
│            └────────────┘                │
│                                          │
│  ┌──────────────────────────────────┐   │
│  │         对话框 (depth:10)        │   │
│  └──────────────────────────────────┘   │
└──────────────────────────────────────────┘
```

**立绘配置表**（`portraitMap`）：

| character | 纹理 key | 位置 | 缩放 | 偏移Y |
|-----------|---------|------|------|-------|
| `bai` | `bai_standing` | center | 1.0 | 0 |
| `xiaolan` | `xiaolan_standing` | center | 0.5 | 0 |
| `qingmiao` | `lingchong_portrait` | center | 0.5 | -80（上移） |
| `player` | null | right | — | — |

**出现动画**：从屏幕外侧滑入 + 淡入（alpha 0→1, duration 400ms）

### 4.3 特写图片（Overlay Image）

```
┌──────────────────────────────────────────┐
```

## 七、序章开发踩坑记录

### 7.1 JS 语法类错误

| 错误 | 原因 | 解决方法 |
|------|------|---------|
| `Uncaught SyntaxError: Unexpected token ','` | class body 中字段定义后用了逗号 `,` 再接方法定义 | class 字段间不用逗号分隔！`;` 也不需要。直接换行即可 |
| `Cannot read properties of undefined (reading 'xxx')` | 访问未初始化的对象属性 | 始终用可选链 `?.` 或先判断存在性 |
| `updateProgress is not defined` | 脚本加载顺序问题，依赖模块尚未初始化 | 确保 `gameState.js` 在 `main.js` 之前加载；确保 `GameData` 在 `IntroScene` 之前就绪 |

### 7.2 数据流不一致 Bug

| 问题 | 现象 | 根因 | 解决方案 |
|------|------|------|---------|
| 左侧卡片显示初始值但右侧详情显示 0 | 《本草情籍》左右数据不同步 | 左侧用 `?? initialValue` fallback，右侧调 `getAttribute()` 返回 `|| 0` | 统一数据源：抽取 `_getAttrValue(attrId, attrDef)` 方法，两侧共用 |
| 属性面板打开时空白 | 点击「📜 情籍」按钮后无内容 | `openModal()` 只做了 DOM 显隐，没触发数据渲染 | 在 `openModal()` 中判断 modalId === 'attributes-modal' 时调用 `updateAttributesUI()` |
| 分类标签无法切换 | 点击「医术能力」「声望交际」等标签无响应 | `bindAttrCategoryEvents()` 在每次 `updateAttributesUI()` 中重复绑定 onclick，且可能被 innerHTML 清掉 | 改用**事件委托**：在 `bindEvents()` 中一次性 `document.addEventListener('click', ...)` 监听标签点击 |

### 7.3 资源加载问题

| 问题 | 现象 | 解决方案 |
|------|------|---------|
| 图片显示为空白/紫色方块 | texture key 未注册或路径错误 | preload 时 `this.load.image(key, path)` 的 key 必须与后面 `setImage()` / `setTexture()` 使用的 key 完全一致 |
| 图片路径 404 | 路径大小写或拼写错误 | Phaser 路径区分大小写！Windows 文件系统不区分但 HTTP/Phaser 加载时区分 |
| 特写图片不显示且 console 有 warn | `textures.exists(imageKey)` 返回 false | 确保在 `preload()` 阶段已注册该 key |

### 7.4 UI 层级（Depth）管理

```
Depth 从低到高：
  7   特写图片（overlayImage）—— 最底层前景元素
  8   角色立绘（characterPortrait）
  10  对话框容器（dialogBox）及其子元素（头像、名字、内容）
  11  对话框内的文字元素
  15  旁白文本（narrationText）/ 继续提示
  20  跳过按钮
  60  奖励提示文字（临时显示，用完即销毁）
```

> **教训**：depth 设错会导致元素被遮挡或遮挡其他元素。特写图片(depth=7)故意设在立绘(depth=8)下方。

### 7.5 动画与状态管理

| 注意事项 | 详情 |
|---------|------|
| 连续同一元素的动画冲突 | 同一 tween 目标若上一个未完成就启动新的，需先用 `tweens.killTweensOf(target)` 清除 |
| 立绘去重优化 | `_currentPortraitCharacter` 记录当前角色ID，相同角色不再重建，避免闪烁 |
| 特写去重优化 | `_currentOverlayImageKey` 记录当前特写key，同图连续多幕跳过动画 |
| 隐藏 vs 销毁 | `_hideOverlayImage()` 采用**立即销毁**（destroy）而非淡出隐藏，避免残留 |
| 场景切换时的清理 | `shutdown()` / `_cleanupSubsystems()` 必须销毁所有创建的 game objects，防止内存泄漏 |

### 7.6 剧情设计经验

1. **nextDelay 取值指南**：
   - `0`：立即下一步（choice、stepAlias）
   - `300~500`：普通对话节奏
   - `1000~1500`：需要阅读时间的内容
   - `2000~3000`：旁白/氛围段落
   - `5000+`：重要情感时刻（慎用，玩家可能不耐烦）

2. **旁白颜色渐变技巧**（S01 开场）：
   - 从深色 `#8a7050` 逐步变亮到 `#f0e8d4`
   - 营造"黑暗中逐渐看清"的氛围感
   - 最后一步设 `triggerFadeIn: true` 触发场景淡入

3. **选择分支设计原则**：
   - 不同选项应有**不同的 NPC 回应文本**，体现选择的后果感
   - 但最终都**汇聚到同一个 nextStep**，避免剧情分支爆炸
   - flag 记录玩家的选择倾向，供后续剧情做差异化

4. **奖励发放时机**：
   - 教学完成后立刻给奖励（正反馈强化学习体验）
   - 重要物品（推荐信、铜钱）放在最后告别场景发放，有仪式感
   - 声望等虚拟数值可在结束时批量处理

5. **教程插入节奏**：
   - 不要连续出太多教程，每 3~5 个对话穿插一个
   - 教程要配合剧情上下文（如"去采集菊花"自然引出采集教学）
   - 教程 hint 文字简洁明确，告知按键和目标
