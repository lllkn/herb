## 1. IntroScene 入口协议（startData）

```js
// 传递给 IntroScene.create() 的参数对象
const startData = {
    debugMode: true,              // 必填：启用调试播放模式（跳场景 + 自动推进）
    debugTargetIdx: sceneIdx,    // 必填：目标场景在 scenes 数组中的索引
    returnToGame: true,          // 必填：剧情播完后返回 GameScene，不显示调试提示
    forceChapter1: chapter1Data, // 推荐：直接传入第一章数据对象，避免 _loadChapter 全局状态歧义
};
```

**IntroScene.create() 的消费逻辑**（`IntroScene.js` 第 262-281 行）：

```js
const initData = this.scene.settings.data;
if (initData && initData.debugMode === true) {
    this._debugMode = true;
    this._debugTargetSceneIdx = initData.debugTargetIdx || 0;
    this._returnToGame = initData.returnToGame || false;
    this._hideHTMLUI();

    if (initData.forceChapter1) {
        this.prologueData = initData.forceChapter1;
        this._isChapter1 = true;
    }
}
// ... 之后 if (!this.prologueData) 才走 _loadChapter / cache.get 兜底加载
```

## 2. 三种调用方式对比

### 2.1 平原大地图 E 键 NPC 交互（正常）

**调用方**：`GameScene.triggerStoryScene(npcData)`  
**代码位置**：`GameScene.js` 第 2066-2101 行

```js
triggerStoryScene(npcData) {
    const chapter1Data = window._chapter1Data;
    // ... 校验 chapter1Data 和 npcData.storyIdx ...

    // ★ 正确：this.scene.start 先 shutdown GameScene，再启动 IntroScene
    this.scene.start('IntroScene', {
        debugMode: true,
        debugTargetIdx: npcData.storyIdx,
        forceChapter1: chapter1Data,
        returnToGame: true
    });
}
```

**✅ 结果**：GameScene 被正确 shutdown → 键盘监听和 update 循环停止 → IntroScene 独享输入 → 剧情正常推进。

### 2.2 翠竹村图片地图点击（已修复）

**调用方**：`GameScene._triggerVillageEvent(sceneIdx, eventName)`  
**代码位置**：`GameScene.js` 第 1412-1436 行

```js
_triggerVillageEvent(sceneIdx, eventName) {
    this.time.delayedCall(150, () => {
        const chapter1Data = window._chapter1Data;
        if (!chapter1Data) { /* ... */ return; }

        const startData = {
            debugMode: true,
            debugTargetIdx: sceneIdx,
            returnToGame: true,
            forceChapter1: chapter1Data,
        };
        window._returnToVillageMap = true;  // ★ 确保返回后是翠竹村地图

        // ★ 修复后：使用 this.scene.start（非 this.game.scene.start）
        this.scene.start('IntroScene', startData);
    });
}
```

**核心修复**：
- **旧代码**：`this.game.scene.start('IntroScene', startData)`  
  → GameScene 不 shutdown，两场景同时运行 → 键盘事件被 GameScene 截获 → 剧情卡死
- **新代码**：`this.scene.start('IntroScene', startData)`  
  → 先 shutdown GameScene → IntroScene 独享输入 → 剧情正常推进

### 2.3 溪流山谷分节触发（新增）

**调用方**：地图组负责在每个子场景结束后触发下一个子场景  
**代码位置**：地图交互层

```js
// 分节 1：玩家进入山谷入口 → 触发 C15a
this.scene.start('C15a', {
    debugMode: true,
    debugTargetIdx: 0,             // C15a 只有 1 个场景，索引固定为 0
    forceChapter1: chapter1Data,
    returnToGame: true
});

// 分节 2：玩家采集完山药 → 触发 C15b
// 地图组监听采集完成事件后：
this.scene.start('C15b', {
    debugMode: true,
    debugTargetIdx: 0,
    forceChapter1: chapter1Data,
    returnToGame: true
});

// 分节 3：玩家跟随青苗指引到达石菖蒲位置 → 触发 C15c
this.scene.start('C15c', {
    debugMode: true,
    debugTargetIdx: 0,
    forceChapter1: chapter1Data,
    returnToGame: true
});

// 分节 4：玩家采集石菖蒲后到达山谷尽头 → 触发 C15d
this.scene.start('C15d', {
    debugMode: true,
    debugTargetIdx: 0,
    forceChapter1: chapter1Data,
    returnToGame: true
});
// C15d 结束后回到 GameScene → 玩家走到 zone_valley_exit → 触发 C16
```

**⚠️ 注意**：
- 每个子场景的 `debugTargetIdx` 都是 `0`（因为 C15a-d 各自只有一个顶层场景）
- 子场景结束时自动返回 GameScene（`returnToGame: true`）
- C16 仍是通过 `zone_valley_exit` 区域触发，保持原有逻辑不变

### 2.4 F3 调试面板触发（参考）

**调用方**：`DebugManager._onTrigger()`  
**代码位置**：`DebugManager.js` 第 399-563 行

```js
_onTrigger() {
    const sceneId = /* ... 从下拉框解析 */;
    const idx = /* ... 在 prologueData.scenes 中查找索引 */;

    // 关闭面板、显示提示...

    // ★ 使用 requestAnimationFrame + setTimeout(50) 推迟到下一帧执行
    const game = this.scene.game;
    requestAnimationFrame(() => {
        setTimeout(() => {
            const introActive = game.scene.isActive('IntroScene');
            if (!introActive) {
                const startData = {
                    debugMode: true,
                    debugTargetIdx: idx,
                    returnToGame: true,
                };
                if (usedFallback && data && data.storyId === 'chapter_1') {
                    startData.forceChapter1 = data;
                }
                game.scene.start('IntroScene', startData);  // ★ 这里用 game.scene.start 是安全的
            } else {
                this.scene._startScene(idx);  // IntroScene 已活跃，直接内部跳
            }
        }, 50);
    });
}
```

**为什么** `game.scene.start` **在这里没问题**：
- F3 的 `_onTrigger` 在 **IntroScene 自身**的上下文中执行（DebugManager 绑在 IntroScene 上）
- 此时 IntroScene 是当前活跃场景，GameScene **早已停止**
- 调用 `game.scene.start('IntroScene', ...)` 会重启 IntroScene，相当于 `this.scene.restart()`，不存在两场景同时运行的冲突

## 3. 关键教训：`this.scene.start` vs `this.game.scene.start`

| API | 行为 | 适用场景 |
|---|---|---|
| `this.scene.start(key, data)` | 先 **shutdown 当前场景**，再启动目标场景 | **推荐**：从当前场景内跳转到另一个场景 |
| `this.game.scene.start(key, data)` | 仅启动目标场景，**当前场景继续运行** | 仅当调用方场景已知已停止，或从全局管理器发起时使用 |

### 核心规则

```
如果调用方需要被销毁（释放输入、update等资源）
    → 用 this.scene.start()

如果调用方场景已经不存在或不应被销毁
    → 用 game.scene.start()
```

## 4. 地图恢复机制：`_returnToVillageMap` → `_forceVillageMap`

翠竹村不是 `triggerStoryScene` 里的 `window._returnPlayerPos`，而是双层 flag 接力：

```
_setupVillageMapClick()
  └→ _triggerVillageEvent()
       └→ window._returnToVillageMap = true      [Step 1: 标记]
            └→ IntroScene 播完 C09/C10/C13
                 └→ 检测 _returnToVillageMap      [Step 2: 中转]
                      └→ window._forceVillageMap = true
                      └→ window._returnToVillageMap = false
                           └→ 回到 GameScene.preload()  [Step 3: 消费]
                                └→ 检测 _forceVillageMap || _returnToVillageMap
                                     └→ this.config.currentMapId = 'village'
```

**双检查兜底**（`GameScene.js` 第 133-139 行）：
```js
// preload() 中同时检查两个 flag，防止 IntroScene 中转分支遗漏
if (window._forceVillageMap || window._returnToVillageMap) {
    this.config.currentMapId = 'village';
    window._forceVillageMap = false;
    window._returnToVillageMap = false;
}
```

## 5. 新增地图交互的 checklist

当需要在新的地图区域添加 NPC/热点触发剧情时，遵循以下步骤：

### Step 1：确定事件 → 场景映射
- 找到 `story_chapter1.json` 中对应的 `sceneIndex`
- 或参考 `DebugManager.EVENT_TO_SCENE` 的事件 ID 命名惯例

### Step 2：确保数据可用
```js
const chapter1Data = window._chapter1Data;
if (!chapter1Data || !chapter1Data.scenes) {
    console.error('第一章剧情数据未加载');
    return;
}
```

### Step 3：构造 startData + 调用
```js
// ✅ 平原大地图 E 键 NPC 模式
this.scene.start('IntroScene', {
    debugMode: true,
    debugTargetIdx: npcData.storyIdx,
    forceChapter1: chapter1Data,
    returnToGame: true
});

// ✅ 翠竹村图片地图点击模式（需要 delayedCall + 地图 flag）
this.time.delayedCall(150, () => {
    window._returnToVillageMap = true;  // 或自定义 map flag
    this.scene.start('IntroScene', startData);
});

// ✅ 溪流山谷分节模式（用场景 ID 直接启动，debugTargetIdx=0）
// 由地图层在采集/到达事件触发时调用
this.scene.start('C15b', {  // 或 C15a / C15c / C15d
    debugMode: true,
    debugTargetIdx: 0,        // 每个子场景内只有一个顶层场景，索引固定为 0
    forceChapter1: chapter1Data,
    returnToGame: true
});
```

### Step 4：确保剧情结束后的地图恢复
- **平原大地图**：IntroScene 自动通过 `window._returnPlayerPos` 恢复位置
- **翠竹村图片地图**：通过 `_returnToVillageMap → _forceVillageMap` 链恢复
- **溪流山谷**：子场景结束后自动返回 GameScene，玩家在地图上继续活动；C16 通过 `zone_valley_exit` 区域触发

## 6. 避免的陷阱

| 陷阱 | 表现 | 修复 |
|---|---|---|
| 在 GameScene 内用 `game.scene.start` 跳 IntroScene | 两场景同时运行，剧情点不动 | 改用 `this.scene.start` |
| 传入 `debugTargetIdx` 为错误索引 | 跳到空白/错误的场景 | 打印核对 `chapter1Data.scenes[idx]`；溪流山谷子场景固定传 `0` |
| `forceChapter1` 未传导致走 `_loadChapter` 分支 | 加载到序章数据而非第一章 | 必须显式传入 `forceChapter1: chapter1Data` |
| 图片地图返回后地图 ID 丢失 | 回到空地图或错误地图 | 添加对应的 `_returnXxxMap` flag |
| C15 拆分后用旧 ID `'C15'` 找不到场景 | 场景不存在 | 改用 `'C15a'` / `'C15b'` / `'C15c'` / `'C15d'` |
