# 2026-05-24 工作记录

## 最终修复：Phaser tilemap 中 GID=0 表示"空瓦片"（行走层碰撞彻底修复）

**经过层层排查，发现三连Bug：**

| Bug# | 问题 | 表现 |
|---|---|---|
| 1 | `data` 是 1D 数组，Phaser 要求 2D | `Cannot read 'index'` 崩溃 |
| 2 | collisionData 用 0=阻挡 + setCollision(0)，但 Phaser 把 GID=0 当空瓦片 | 所有碰撞瓦片被跳过，碰撞层空 |
| 3 | 之前 fix#2 改 `-1`=空 但 -1 非标准值，且 firstgid=0 时 GID=0 仍是空 | 依然不生效 |

**最终方案：** `addTilesetImage` 设 `firstgid=1` → collisionData 用 `1`=阻挡, `0`=通行 → `setCollision(1)`
- GID=1 在 tileset 中有有效瓦片 ✅
- GID=0 是 Phaser 原生空瓦片 ✅
- 碰撞 layer 不可见 ✅

**两条加载路径均已统一：** 都通过 `this._rawMapJson` + `setupWalkCollision()` 无参调用

**根因：** `_collision_tile` 纹理只有 1 块瓦片（16×16/16×16=1），`addTilesetImage` 默认 `firstgid=0`，tileset 只有 gid=0 有效。但 collisionData 用 1 表示阻挡 + `setCollision(1)`，gid=1 在 tileset 中不存在 → 碰撞完全不生效。

**修复：**
- collisionData: 0=阻挡（匹配唯一瓦片gid=0），-1=通行（Phaser空瓦片）
- `setCollision(0)` 替代 `setCollision(1)`

## 修复 load.json 与 tilemapTiledJSON 加载冲突导致绿屏

**根因：** `load.json('gameMapRaw')` 和 `load.tilemapTiledJSON('gameMap')` 同时加载同一个 JSON 文件，Phaser loader 内部冲突导致 tilemap 数据被破坏，所有瓦片渲染失败 → 绿屏。

**修复：** 放弃 `load.json`，改为在 `initTiledMap()` 中 **`make.tilemap` 之前** 直接 `cache.json.get('gameMap')` 存到 `this._rawMapJson`，`setupWalkCollision` 从 `this._rawMapJson` 读取。

## 修复 initTiledMap 路径下行走层碰撞不生效（溪流地图）

**根因：** `load.tilemapTiledJSON` 把数据存入 **tilemap cache**（`cache.tilemap`），不走 **JSON cache**。`setupWalkCollision()` 用 `cache.json.get('gameMap')` 读不到数据，`rawJson` 为 null，直接 return 跳过碰撞。fetch fallback 路径因为传了 `mapData` 不受影响。

**修复：**
1. `preload()` 新增 `this.load.json('gameMapRaw', mapConfig.jsonPath)`，独立加载一份原始 JSON 到 JSON cache
2. `setupWalkCollision()` 改为读 `gameMapRaw`（`rawJsonOverride || this.cache.json.get('gameMapRaw')`）
3. `tryLoadMap()` 缓存检查同时支持 `gameMap` 和 `gameMapRaw` 两个 key
4. `preload()` 清理旧缓存时同时处理两个 key

## 修复 setupWalkCollision 导致 Phaser 崩溃（Cannot read 'index'）

**根因：** `make.tilemap({ data: [...] })` 中 `data` 必须是 **2D 数组**（每行一个子数组），传 1D 数组会导致 Tilemap Layer 结构异常，物理碰撞时报 `Cannot read properties of undefined (reading 'index')`

**修复：**
1. `setupWalkCollision()` 改为生成 2D collisionData（双层 for 循环）
2. 抽出 `bindWalkCollider()` 方法，带 `_walkColliderBound` 防重复 flag，所有碰撞绑定统一走这里
3. 原来三处 `physics.add.collider` 调用统一替换为 `bindWalkCollider()`

## 修复 IntroScene 剧情数据加载失败（显示"备用剧情"）

**问题根因：**
- `BootScene.preload()` 加载 `prologue_data` 到 Phaser 缓存
- `scene.start('IntroScene')` 切换场景后，`IntroScene.preload()` 再次 `load.json('prologue_data', ...)` 
- Phaser 发现缓存中已有同 key，跳过加载，但场景切换时缓存被释放，导致 `create()` 里 `cache.json.get('prologue_data')` 返回 null

**修复方案：**
1. `IntroScene.preload()` 先清除旧缓存再加载（`cache.json.remove` + `load.json`）
2. `create()` 改为：先查缓存，缓存有则直接用，缓存无则用 fetch 兜底加载
3. 抽出 `_initScene()` 方法，数据就绪后统一调用（支持同步和异步两条路径）

## 修复行走层碰撞（玩家只能在行走层移动）

**问题根因（两个 bug）：**
1. `create()` 中先调 `tryLoadMap()` → `initTiledMap()` → `setupWalkCollision()`，但此时 `createPlayer()` 还未执行，`this.player` 为 null，导致 `physics.add.collider` 没绑上
2. fetch fallback 路径 `createMapFromData()` 根本没有调用 `setupWalkCollision()`，且即使调了，因为不经过 Phaser 缓存，`this.cache.json.get('gameMap')` 会返回 null

**修复方案：**
1. `setupWalkCollision()` 新增可选参数 `rawJsonOverride`，fetch 路径直接传 mapData，避免缓存问题
2. `initTiledMap()` 内的 `setupWalkCollision()` 只准备 collisionLayer，不再绑定玩家碰撞
3. `create()` 中在 `createPlayer()` 之后统一绑定碰撞：`physics.add.collider(this.player, this.collisionLayer)`
4. `createMapFromData()` 末尾补充调用 `setupWalkCollision(mapData)` 并绑定碰撞
